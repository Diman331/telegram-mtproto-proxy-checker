# Telegram MTProto Proxy Checker Bot — Проект

**Репозиторий:** https://github.com/Diman331/telegram-mtproto-proxy-checker

## Описание

Телеграм-бот для проверки MTProto прокси Telegram на работоспособность с измерением задержки.

## Основной функционал

### Проверка прокси
- Парсинг ссылок из текста (tg://proxy? и https://t.me/proxy?)
- Поддержка пересланных сообщений из каналов
- Пакетная проверка нескольких прокси сразу
- Измерение задержки (ping) в миллисекундах
- Прогресс-бар при проверке >10 прокси

### Форматы прокси
- `tg://proxy?server=...&port=...&secret=...`
- `https://t.me/proxy?server=...&port=...&secret=...`
- `https://t.me/+proxy?server=...&port=...&secret=...`

### База данных прокси
- Все проверенные прокси сохраняются в `proxies_db.json`
- Хранится статус (working/failed), задержка, время проверки
- Автоматическая очистка нерабочих прокси каждые 2 проверки

### Автопроверка
- Планировщик проверяет базу по расписанию
- Интервал: 6-168 часов (настраивается)
- Отчёт отправляется админу в Telegram
- Настройки в `bot_settings.json`

## Технические детали

### TDLib
- Используется `prebuilt-tdlib` для кроссплатформенности
- Библиотека `libtdjson.so` подключается через симлинк в `/usr/lib/`
- Проверка через `addProxy` + `pingProxy`

### Хранение данных
- Прокси хранятся как объект: `{ "url": { checked_at, status, latency } }`
- Счётчик проверок для очистки (каждые 2 проверки)
- Автопроверка использует `setTimeout` с пересчётом интервала

### Обработка ошибок
- Markdown → HTML (проблемы с `_` в URL)
- Таймаут 15 секунд на прокси
- Пакетная проверка по 10 прокси для избежания таймаута Telegram (90 сек)

## Админ-команды

Требуют `ADMIN_ID` в `.env`:
- `/settings` — управление автопроверкой
- `/chkdb` — проверка всей базы

Если `ADMIN_ID` не установлен — команды доступны всем (с предупреждением в консоль).
